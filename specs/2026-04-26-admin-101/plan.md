# Plan

## 1. 盘点与映射

- 目标：确认保留页、删除页、共用布局、受影响导航与路由。
- 涉及的改动面：路由树、侧边栏、顶部导航、命令菜单、认证流程入口。
- 完成定义：保留集合和移除集合明确，不再存在“先留着以后再看”的模板入口。

## 2. 路由清理

- 目标：删除或下线无关路由，只保留 `Sign In`、`OTP`、受保护壳层、`Dashboard`、`Tasks`、`Logs`。
- 涉及的改动面：`src/routes`、路由注册、受保护路由入口。
- 完成定义：无关主路由不再可达，也不再参与主流程。

## 3. 导航清理

- 目标：收敛侧边栏、顶部导航、命令菜单、搜索项，移除模板业务词汇。
- 涉及的改动面：导航数据、全局检索项、页面级导航、用户菜单中的无关入口。
- 完成定义：后台导航只服务 `Dashboard`、`Tasks`、`Logs` 和认证主流程。

## 4. 页面骨架收敛

- 目标：将 `Dashboard`、`Tasks`、`Logs` 改为统一业务化占位页。
- 涉及的改动面：页面标题、说明文案、空态、卡片区、演示数据组件。
- 完成定义：页面不再出现销售、聊天、用户管理或 SaaS 模板内容。

## 5. 认证入口收敛

- 目标：只保留 `Sign In + OTP` 主流程，移除其余演示 auth 页面在主流程中的暴露。
- 涉及的改动面：登录页文案、认证路由入口、跳转关系、导航可见性。
- 完成定义：用户从主流程只能进入 `Sign In` 与 `OTP`，不会误入其他 auth demo。

## 6. 文案与残留检查

- 目标：全局清除与使命不一致的模板名称、公司名、SaaS 文案、演示 badge 和无关说明。
- 涉及的改动面：页面文案、侧边栏团队信息、按钮标签、空态说明。
- 完成定义：首次打开后台时，用户不会把它识别成通用 SaaS 模板。
