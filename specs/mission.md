# Swan-Admin Mission

## 定位

Swan-Admin 是面向数字货币量化系统的内部运营控制台，用于帮助管理员、运营、风控和技术支持团队统一查看系统状态、市场行情、资产概况、任务执行与日志审计信息。

Swan-Admin v1 的核心目标不是提供完整交易终端，也不是面向终端投资者提供资产门户，而是建设一个可监控、可理解、可追踪、可介入的管理后台。

## 目标用户

- 管理员：负责后台整体配置、菜单管理、权限分配与日常运营。
- 运营人员：关注市场状态、账户资产、任务执行和异常处理。
- 风控人员：关注风险暴露、异常波动、系统告警和审计轨迹。
- 技术支持：关注运行状态、日志排查、故障定位和恢复辅助。

## 核心价值

- 市场可见性：让内部团队快速看到关键市场数据与异常变化。
- 资产可理解性：让用户清楚当前资产规模、结构、波动和关键变化。
- 运行可追踪性：让任务、日志、异常和操作记录可以被查询和解释。
- 处置可介入性：让后台在风险和异常场景下支持快速定位和人为干预。
- 使用舒适性：让高频后台操作保持低认知负担、清晰反馈和稳定体验。

## V1 范围

Swan-Admin v1 聚焦以下能力：

- 行情总览：提供类似交易所风格的实时或准实时市场概览页面。
- 资产看板：展示资产总览、账户维度分布、关键变动和趋势信息。
- 日志审计：支持系统日志、操作日志与异常事件查询。
- 异常可见性：突出风险提示、失败状态、异常波动和关键告警入口。
- 基础任务监控：展示任务状态、执行结果、失败原因和处理入口。

## 非目标

以下内容不属于 Swan-Admin v1 的核心交付：

- 完整的交易终端、下单工作台或专业交易员操作界面。
- 复杂策略研究平台、回测平台或量化研究工作台。
- 面向终端投资者的门户、收益展示站点或客户自助服务台。
- 后端服务架构、行情服务实现、数据库模型或全局 API 宪章。

## 产品原则

### 可观测性与风控优先

任何核心功能都应优先满足以下要求：

- 关键状态可见。
- 异常原因可追踪。
- 风险信息可突出。
- 历史行为可审计。

### 后台体验舒适且可信

“舒适、贴心”的后台体验在本项目中具体表现为：

- 低认知负担：信息分层清晰，避免噪音与模板堆砌。
- 关键风险突出：盈利信息和风险信息必须同时可见，不能只强调增长。
- 操作可恢复：高风险动作要有明确确认、撤销说明或回退路径。
- 状态可解释：加载、空态、失败态、延迟态都要有明确表达。
- 反馈一致：数值、图表、状态标签、提示语保持统一语义。

## V1 成功标准

若达到以下结果，可视为 Swan-Admin v1 达成目标：

- 内部用户可以在一个后台内完成市场观察、资产查看、日志排查和基础任务监控。
- 首页和核心页面能够在短时间内回答“市场发生了什么”“资产现在怎样”“最近出了什么异常”。
- 风险与异常信息不再隐藏在二级页面或零散日志中，而能被主动发现。
- 现有模板后台被收敛为量化系统语义，减少无关模块和命名噪音。
